# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project overview

Semantic segmentation system for detecting rice field lodging and disease (Black Stripe Dwarfism) from images. Uses a DeepLabv3+ encoder-decoder architecture with custom attention mechanisms (CGAFusion, LocalAttention, TBFE).

**Classes:** `_background_`, `Black_stripe_dwarfism`, `lodging`

## Environment

No build system or linter is configured. Install dependencies:

```bash
pip install -r requirements.txt
pip install einops onnx onnx-simplifier
```

## Key commands

### Train

1. Prepare VOC-format dataset: `VOCdevkit/VOC2007/JPEGImages/` (`.jpg`) and `VOCdevkit/VOC2007/SegmentationClass/` (`.png` masks, pixel values = class index starting at 0).
2. Configure hyperparameters at the top of `train.py` (inside `if __name__ == "__main__":`): `Cuda`, `num_classes`, `backbone` (`"mobilenet"` or `"xception"`), `VOCdevkit_path`, `model_path`, `Freeze_Train`, batch size, epochs, etc.
3. Run:
   ```bash
   python train.py
   ```
4. Multi-GPU DDP (Linux only; Windows uses DataParallel automatically):
   ```bash
   CUDA_VISIBLE_DEVICES=0,1 python -m torch.distributed.launch --nproc_per_node=2 train.py
   ```

### Predict

Set `mode` in `predict.py` to one of: `"predict"` (single image), `"dir_predict"` (batch directory), `"video"`, `"fps"`, `"export_onnx"`. Configure the model path, number of classes, and backbone in `deeplab.py`'s `_defaults` dict. Then:

```bash
python predict.py
```

### Evaluate (mIoU)

Configure `num_classes`, `name_classes`, and `miou_mode` at the top of `get_miou.py`, then:

```bash
python get_miou.py
```

Generates confusion matrices, mIoU/mPA/Recall/Precision plots in `miou_out/`.

### Generate VOC dataset splits

```bash
python voc_annotation.py
```

## Architecture

### Active model: `nets/cga.py`

Encoder-decoder with skip connections, imported as `DeepLab` by both `train.py` and `deeplab.py`.

**Encoder:**
- Backbone (MobileNetV2 or Xception), outputs features at 1/16 input resolution
- ASPP: 4 parallel dilated convolutions (rates 1, 6, 12, 18) + global average pooling
- **CGAFusion** (Cross-Gated Attention Fusion, from `nets/attention.py`) connects ASPP branches — combines spatial, channel, and pixel attention

**Decoder:**
- **LocalAttention** on low-level backbone features (soft-pooling → conv → sigmoid gating)
- Skip connection: low-level features pass through 1×1 conv (48 channels), concatenated with upsampled ASPP output
- Two 3×3 convs (256 ch) with dropout (0.5, 0.1), then 1×1 → `num_classes`, bilinear upsampled to input size

### Custom attention modules (`nets/attention.py`)

- `SpatialAttention` — avg/max pool across channels → 7×7 conv → spatial map
- `ChannelAttention` — global avg pool → MLP (reduction ratio 8) → channel weights
- `PixelAttention` — concatenate two feature maps → grouped channel-wise conv → sigmoid
- `CGAFusion` — combines spatial + channel + pixel attention for fusing ASPP branches
- `TBFE` — 1×1 conv + depthwise separable conv + 3D conv branches with residual
- `LocalAttention` — soft-pool downsample → conv → sigmoid gate for local importance
- `SoftPooling2D` — softmax-based pooling alternative

### Loss functions (`nets/deeplabv3_training.py`)

Composite loss controlled by flags in `train.py`:
- `CE_Loss` — standard cross-entropy with class weights and ignore index
- `Focal_Loss` — gamma=2, alpha=0.5 (for class imbalance)
- `Dice_loss` — Sørensen-Dice coefficient based

### Training pipeline

- **Freeze phase:** Optionally freeze backbone for initial epochs (`Freeze_Train = True` in `train.py`)
- **Optimizer:** SGD with Nesterov momentum (default), configurable batch size (LR auto-scales by `batch_size / 16`)
- **LR schedule:** Cosine annealing (default) or step decay via `get_lr_scheduler()`
- **Mixed precision:** fp16 via `torch.cuda.amp.GradScaler` (requires PyTorch ≥1.7.1)
- **DDP:** Supported via `torch.distributed.launch` (Linux); Windows falls back to DataParallel
- **Data augmentation** (`utils/dataloader.py`): random scaling (0.25×–2×), aspect ratio distortion, horizontal flip, translation/crop, Gaussian blur (25%), rotation (±10°), HSV jitter
- Weights saved to `logs/` with timestamped subdirectories

### Data format

PASCAL VOC structure:
```
VOCdevkit/VOC2007/
  JPEGImages/         # .jpg input images (any size, auto-resized)
  SegmentationClass/  # .png labels — pixel value = class index (0 = background, 1 = class 1, ...)
  ImageSets/Segmentation/
    train.txt         # generated by voc_annotation.py
    val.txt
```

**Critical:** Label pixels must use contiguous class indices starting at 0. A common mistake is using 0 for background and 255 for targets — this trains but predictions will be wrong. Use 0, 1, 2, ... for each class.

## Configuration pattern

There are no YAML/JSON config files. All settings are module-level variables at the top of each entry-point script:

| Script | Where to configure |
|--------|--------------------|
| `train.py` | Variables inside `if __name__ == "__main__":` block |
| `deeplab.py` | `DeeplabV3._defaults` dict |
| `get_miou.py` | Top-level variables: `miou_mode`, `num_classes`, `name_classes` |
| `predict.py` | Top-level: `mode`, `count`, `name_classes`, `video_path`, etc. |

The active model import is in `train.py` line 12 and `deeplab.py` line 12 — both import from `nets.cga`. To switch model variants, change these imports.
